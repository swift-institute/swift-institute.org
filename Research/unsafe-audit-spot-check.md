## Notes

### Agent 1 — S3 misclassification detail

`Witness.Values._Storage` was classified as Cat A (synchronized) but the class contains zero synchronization primitives. The outer `Witness.Values` struct uses CoW semantics for mutation safety, but CoW is not thread-synchronization — it is a value-semantic sharing optimization. The `_Storage` class is simply a reference-type dictionary holder where `@unchecked Sendable` is required because `UnsafeRawPointer` values block structural inference. This is Cat D.

Recommendation: Audit Agent 1's remaining 14 Cat A classifications for similar CoW-is-synchronization conflation. The other two Agent 1 samples (S1, S2) were correctly classified as Cat A with genuine synchronization primitives, so this may be an isolated error.

### Agent 2 — S6 factual error on value-generic

Agent 2 flagged `Stack.Bounded` as a D-candidate for `<let N: Int>`, but `Stack.Bounded` does not have a value-generic parameter. It uses runtime `requestedCapacity: Index.Count`. The primary B classification is correct, but the D-candidate reasoning contains a factual error. This does not change the verdict (the B classification stands) but suggests Agent 2 may have confused `Stack.Bounded` with `Stack.Static` or similar.

### B/D boundary ambiguity

Three samples (S4, S9, S10) landed on the B/D boundary. The ecosystem has a genuine dual-classification pattern where `~Copyable` types with `@_rawLayout` or phantom generics have both ownership-transfer (B) and structural-inference-gap (D) characteristics. The handoff's D-queue mechanism correctly anticipates this — these are exactly the cases that need principal adjudication.
